原子性
\n原子性要求，事务是一个不可分割的执行单元，事务中的所有操作要么全都执行，要么全都不执行。
\n一致性
\n一致性要求，事务在开始前和结束后，数据库的完整性约束没有被破坏。
\n隔离性
\n事务的执行是相互独立的，它们不会相互干扰，一个事务不会看到另一个正在运行过程中的事务的数据。
\n持久性
